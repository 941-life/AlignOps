# 🎯 AlignOps 사용자 플로우 재설계 (심사자 관점)

## 현재 문제점

### ❌ 현재 상태
1. **빈 대시보드**: 처음 들어가면 데이터가 없어서 뭘 해야 할지 모름
2. **복잡한 네비게이션**: Datasets, Control Plane 등 용어가 어려움
3. **벡터 DB 시각화 부족**: Lineage 페이지에만 간단히 있음
4. **데모 경험 불가**: 데이터를 직접 만들어야 체험 가능

### ✅ 개선 방향

## 이상적인 심사자 플로우

```
1단계: 랜딩 (5초 이내 핵심 파악)
   ↓
   → "데모 보기" 버튼 클릭
   ↓
2단계: 자동 데모 데이터 로딩 (10초)
   ↓
   → 실시간으로 v1, v2 생성 과정 표시
   ↓
3단계: 대시보드 (데이터 있는 상태)
   ↓
   → demo_vlm_dataset 클릭
   ↓
4단계: 버전 비교 뷰
   ↓
   → "Drift 분석 보기" 클릭
   ↓
5단계: 인터랙티브 벡터 시각화
   ↓
   → v1 (자연) vs v2 (도시) 분포 차이
   → 문제 샘플 하이라이트
   ↓
6단계: Gemini 판단 보기
   ↓
   → 왜 WARN/BLOCK 인지 이해
   ↓
완료: 핵심 가치 이해
```

## 필요한 개선사항

### 1. 랜딩 페이지 개선
- [ ] Hero 섹션 추가 (무엇을 하는 서비스인지)
- [ ] "Try Demo" 버튼 추가
- [ ] 빈 상태일 때 온보딩 가이드

### 2. 원클릭 데모 기능
- [ ] 프론트엔드에서 데모 데이터 자동 생성
- [ ] 진행 상황 실시간 표시
- [ ] 완료 후 자동 네비게이션

### 3. 벡터 시각화 강화
- [ ] 인터랙티브 2D scatter plot
- [ ] v1 vs v2 분포 비교
- [ ] 호버 시 이미지 프리뷰
- [ ] 문제 샘플 하이라이트

### 4. 간소화된 네비게이션
- [ ] "Demo" 탭 추가
- [ ] 용어 쉽게 변경 (Control Plane → Settings)
- [ ] 각 페이지에 도움말 툴팁

### 5. 스토리텔링 강화
- [ ] 각 단계마다 설명 추가
- [ ] "Next: 다음 단계" 버튼
- [ ] 진행 상황 표시 (1/5 단계)

## 구현 우선순위

### P0 (필수)
1. **랜딩 페이지에 데모 버튼**
2. **원클릭 데모 생성 기능**
3. **벡터 시각화 (scatter plot)**

### P1 (중요)
4. 온보딩 가이드
5. 네비게이션 간소화

### P2 (선택)
6. 인터랙티브 튜토리얼
7. 비디오 가이드

## 구현 계획

### Phase 1: Quick Wins (30분)
- 랜딩 페이지에 Hero 섹션
- "Load Demo Data" 버튼
- 빈 상태 개선

### Phase 2: Core Experience (1시간)
- 벡터 시각화 (Recharts scatter plot)
- 데모 데이터 자동 생성
- 진행 상황 표시

### Phase 3: Polish (30분)
- 온보딩 툴팁
- 네비게이션 개선
- 도움말 추가
